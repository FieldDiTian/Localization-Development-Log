“门控”就是 **gate / gating**，意思是：先设定一组质量阈值，数据进来后先检查，**通过才使用，不通过就丢弃或降级处理**。

在这个 README 里，主要指 **RTK 质量门控**：

- Atlas INS 输出 `/gps_p1/filtered_odom`
    
- 定位器读取其中的 `pose.covariance`
    
- 如果位置协方差小于阈值，认为 RTK/INS 质量可靠，就用于初始化或恢复定位
    
- 如果协方差超过阈值，认为这帧不可靠，就丢弃，不拿它来重置位姿
    

例如：
```text
max_pose_var_xy
max_pose_var_z
```
就是 XY 平面和 Z 方向允许的最大位置方差阈值。

通俗说：  
**门控 = 质量检查关卡。过关的数据才能参与定位，没过关的数据不能用。**
