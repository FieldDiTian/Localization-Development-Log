录制了GLIM的前端demo
现在的online adapter同时消耗pcap中的IMU p1 time来计算时间，然后用mcap中的剩余信息

这说明 driver/recording 把 burst arrival pattern 写进了 `header.stamp`。
**不能直接用原始 MCAP IMU stamp 做精确 IMU preintegration / deskew / live GICP propagation**。
- **如果要真实 IMU sample time，只能从同次 INS PCAP 解析 `IMU_OUTPUT.p1_time`**，因为 MCAP 的 `sensor_msgs/Imu` 已经没有这个字段。


```mermaid
flowchart LR
  MCAP["MCAP replay<br/>/atlas/pose_filtered<br/>/dlio_raw/luminar_*<br/>/clock"]
  PCAP["INS PCAP replay<br/>FusionEngine IMU_OUTPUT<br/>真实 p1_time"]
  CFG["初始信息<br/>PCD map<br/>T_world_utm.txt<br/>UTM origin<br/>av24.urdf<br/>localization.yaml"]

  MCAP --> POSE_RAW["/atlas/pose_filtered"]
  MCAP --> LIDAR_RAW["/dlio_raw/luminar_front|left|right/points"]
  PCAP --> IMU_RAW["/atlas/imu_calibrated"]

  POSE_RAW --> ADAPTER["dlio_input_adapter_node<br/>pose p1_time -> ROS stamp<br/>IMU retime<br/>LiDAR bridge"]
  IMU_RAW --> ADAPTER
  LIDAR_RAW --> ADAPTER
  CFG --> ADAPTER

  ADAPTER --> IMU["/gps_p1/imu"]
  ADAPTER --> GT["/gps_p1/filtered_odom_map"]
  ADAPTER --> ODOM["/gps_p1/filtered_odom<br/>/gps_p1/filtered_odom_rtk_fixed"]
  ADAPTER --> LIDAR["/luminar_front|left|right/points"]

  CFG --> TF["robot_state_publisher<br/>/tf_static from av24.urdf"]

  IMU --> GICP["gicp_localization_node<br/>scan-to-map GICP<br/>deskew / init / recovery"]
  GT --> GICP
  LIDAR --> GICP
  TF --> GICP
  CFG --> GICP

  GICP --> OUT1["/gicp/localization/odom"]
  GICP --> OUT2["/gicp/localization/pose"]
  GICP --> OUT3["/gicp/localization/path"]
  GICP --> OUT4["/gicp/localization/map"]
  GICP --> DBG["debug topics<br/>fitness / hessian / gt error"]

  OUT1 --> RVIZ["RViz"]
  OUT4 --> RVIZ
  DBG --> RVIZ

  OUT1 --> MON["live_gnss_error_monitor.py"]
  GT --> MON

  OUT1 --> REC["ros2 bag record / eval"]
```


```mermaid
flowchart TD
  S["启动"] --> P["读取 launch / YAML 参数"]
  P --> M["加载 PCD map"]
  P --> U["读取 T_world_utm / UTM origin"]
  P --> W["等待 IMU / LiDAR / GT odom"]

  W --> I["IMU callback<br/>缓存 IMU buffer<br/>预测 odom"]
  W --> O["GT odom callback<br/>RTK covariance gate<br/>缓存 gt_odom_buffer"]
  O --> INIT{"是否已有初始 pose?"}
  INIT -- "否" --> SEED["用第一帧有效 GT odom 初始化"]
  INIT -- "是" --> KEEP["作为 cross-check / recovery / error reference"]

  W --> L["LiDAR callback<br/>front scan 到达"]
  L --> C{"lidar_concat enabled?"}
  C -- "是" --> AUX["拼接 left / right aux scan"]
  C -- "否" --> PRE["只用 front scan"]
  AUX --> PRE

  PRE --> D{"deskew enabled?"}
  D -- "是" --> DESKEW["用 IMU buffer 去畸变"]
  D -- "否" --> VOX["crop / voxel / filter"]
  DESKEW --> VOX

  VOX --> GUESS["用上一帧 / IMU observer 生成 initial guess"]
  GUESS --> ALIGN["nano_gicp align scan to map"]
  ALIGN --> METRIC["计算 fitness / delta / hessian condition"]
  METRIC --> OK{"通过质量门限?"}

  OK -- "是" --> UPDATE["接受配准<br/>更新 pose / observer state"]
  OK -- "否" --> REJECT["拒绝本帧<br/>沿预测状态走"]
  REJECT --> SNAP{"连续失败且 GT 可用?"}
  SNAP -- "是" --> RECOVER["snap / recover 到 RTK-gated GT"]
  SNAP -- "否" --> PUB["发布结果"]
  UPDATE --> PUB
  RECOVER --> PUB

  PUB --> OUT["/gicp/localization/odom<br/>/pose<br/>/path<br/>/map<br/>debug topics"]
```
