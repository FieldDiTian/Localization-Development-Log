**结论**
你负责的边界可以这样看：localization 输出车的“自车状态”，perception 输出“对手车/障碍物状态”。

Localization 的主输出是 `/vehicle/state` 和 `/localization/global/odom`；perception 的主输出是 `/tracked_objects`。下游 RDE/RPE/RVC/MPC 都围绕这两个接口走。

**Localization 数据流**
车端 tmux 会启动 GPS/INS 和 localization，见 [iac_car.yaml](/home/roar/Documents/race_common/tools/tmux_configs/iac_car.yaml:38)。

入口：

```text
NovAtel A -> /gps_na/filtered_odom, /gps_na/odom, /gps_na/imu
Atlas     -> /gps_p1/filtered_odom, /gps_p1/imu
Steering  -> /vehicle/steering_report
```

VKS 在 [vehicle_kinematic_state.launch.py](/home/roar/Documents/race_common/src/localization/vehicle_kinematic_state/launch/vehicle_kinematic_state.launch.py:50) 里把这些 remap 进去，输出：

```text
/vks/filtered_odom
/vks/imu
/vks/gnss_odom
/vehicle/state
/vehicle/navsatfix
/vks/telemetry
```

然后 robot_localization 在 [robot_localization_interface.launch.py](/home/roar/Documents/race_common/src/localization/robot_localization_interface/launch/robot_localization_interface.launch.py:37) 消费：

```text
/vks/filtered_odom -> odom0
/vks/imu           -> imu0
```

输出：

```text
/localization/global/odom
/localization/global/accel
/localization/global/diagnostics
```

注意：`localization_all.launch.py` 只启动 EKF，见 [localization_all.launch.py](/home/roar/Documents/race_common/src/launch/localization_launch/launch/localization_all.launch.py:32)。`/vehicle/state` 是 VKS 产的，VKS 在 autonomy launch 里启动，或者你单独启动 `vehicle_kinematic_state.launch.py`。

Lidar 配置在 [iris.param.yaml](/home/roar/Documents/race_common/src/launch/iac_launch/param/lidar_param/iris.param.yaml:4)，三颗点云入口应是：


legacy perception launch 也正是消费这三个 topic，见 [perception_lidaronly.launch.py](/home/roar/Documents/race_common/src/launch/perception_launch/launch/perception_lidaronly.launch.py:42)。它期望中间输出：


Hybrid 模式会额外走 camera/yolo，见 [perception_hybrid.launch.py](/home/roar/Documents/race_common/src/launch/perception_launch/launch/perception_hybrid.launch.py:42)，中间融合输出是：

**当前 perception 主要卡点**
仓库里还有一套 [autoware_perception_launch](/home/roar/Documents/race_common/src/perception/autoware_perception_launch/launch/autoware_perception.launch.py:24)，但它默认输入是：

```text
/luminar_front_points
/luminar_left_points
/luminar_right_points
```

这和车端实际的 `/luminar_front/points` 命名不一致。并且它依赖的包名还是老接口，比如 `pointcloud_preprocessor`、`multi_object_tracker`，当前 Jazzy/Autoware 源码里很多包已经是 `autoware_*` 前缀。所以 perception 不能只靠打开 `LAUNCH_PERCEPTION=True`，需要先统一 launch 方案和 topic/type contract。

**你应该重点验这些**
```bash
source install/setup.bash

ros2 topic hz /gps_na/filtered_odom
ros2 topic hz /gps_p1/filtered_odom
ros2 topic hz /vks/filtered_odom
ros2 topic hz /vehicle/state
ros2 topic hz /localization/global/odom

ros2 topic hz /luminar_front/points
ros2 topic hz /luminar_left/points
ros2 topic hz /luminar_right/points
ros2 topic list | grep -E 'vimba|radar|tracked_objects'
```

最关键的工程约定：localization 必须稳定产 `/vehicle/state`；perception 最终必须稳定产 `/tracked_objects`，而且类型要能被 RDE/RPE 消费。RDE 明确把 `/vehicle/state` 当 localization 输入、`/tracked_objects` 当对手车输入，见 [race_decision_engine.launch.py](/home/roar/Documents/race_common/src/planning/race_decision_engine/launch/race_decision_engine.launch.py:67)。
